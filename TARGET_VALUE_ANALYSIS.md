### **2. Progress Calculation** üìà

**Code:**
```typescript
// lib/database.ts - Calculate progress percentage
async getActiveGoals(userId: string) {
  const { data } = await supabase
    .from('goal')
    .select('*')
    .eq('user_id', userId)
    .eq('is_active', true);

  return data?.map(goal => {
    const currentValue = goal.current_value || 0;
    const targetValue = goal.target_value || 1;  // ‚úÖ Read from database
    
    // Calculate progress as percentage
    const progress = Math.round((currentValue / targetValue) * 100);
    
    return {
      ...goal,
      progress: progress,  // 0-100%
    };
  });
}
```

**Example:**
```
Database:
- target_value: 10
- current_value: 7

Calculation:
progress = (7 / 10) √ó 100 = 70%

UI shows: "7/10 matches (70%)"
```

### **3. Goal Completion Check** ‚úÖ

**Code:**
```typescript
// lib/database.ts - Check if goal is complete
async updateGoalProgress(goalId: string, currentValue: number) {
  // Fetch the target_value from database
  const { data: goal } = await supabase
    .from('goal')
    .select('target_value')  // ‚úÖ Get target from DB
    .eq('goal_id', goalId)
    .single();

  // Check if goal is complete
  const isCompleted = currentValue >= (goal?.target_value || 0);  // ‚úÖ Compare
  
  // Update the goal
  await supabase
    .from('goal')
    .update({ 
      current_value: currentValue,
      is_completed: isCompleted  // ‚úÖ Mark complete if reached
    })
    .eq('goal_id', goalId);
}
```

**Example:**
```
Match completes:
- Current: 9 matches
- After match: 10 matches
- Target: 10 matches

Check: 10 >= 10 ‚Üí TRUE
Update: is_completed = true

UI shows: "GOAL COMPLETE! üéâ"
```

async updateGoalsAfterMatch(userId, matchResult, finalScore, opponentScore) {
  // Get all active goals
  const activeGoals = await this.getActiveGoals(userId);

}

## ‚úÖ Do We Need to Track `target_value`?

### **YES! Absolutely Critical** ‚≠ê

**Reasons:**

#### **1. Progress Calculation** üìä
```typescript
progress = (current_value / target_value) √ó 100
```
Without `target_value`, we can't show progress percentage!

#### **2. Goal Completion** ‚úÖ
```typescript
is_completed = current_value >= target_value
```
Without `target_value`, we can't determine when goal is achieved!

#### **3. Visual Display** üé®
```
UI shows: "7/10 matches"
                ^^ target_value needed!
```

#### **4. Multiple Goals** üéØ
Different users have different targets:
- User A: "Play 10 matches"
- User B: "Play 50 matches"
- User C: "Play 5 matches"

Each needs their own `target_value` stored separately.

#### **5. Goal History** üìú
When we look back at completed goals:
```
"You completed: Play 10 matches in 1 Month"
                      ^^ need to remember the target
```

## üìä Database Queries Using `target_value`

### **Query 1: Get Active Goals with Progress**
```sql
SELECT 
  goal_id,
  category,
  target_value,
  current_value,
  ROUND((current_value::float / target_value) * 100) as progress_percent,
  CASE 
    WHEN current_value >= target_value THEN true 
    ELSE false 
  END as is_complete
FROM goal
WHERE user_id = $1 
  AND is_active = true;
```

### **Query 2: Find Completed Goals**
```sql
SELECT *
FROM goal
WHERE user_id = $1
  AND current_value >= target_value;
```

### **Query 3: Goals Close to Completion**
```sql
SELECT *
FROM goal
WHERE user_id = $1
  AND is_active = true
  AND (current_value::float / target_value) >= 0.8  -- 80%+ complete
ORDER BY (target_value - current_value) ASC;
```

## üîÑ Can `target_value` Change?

### **Current Implementation:** ‚ùå No Update Function

Right now, once a goal is created, `target_value` is fixed.

**Code Check:**
```typescript
// ‚úÖ We have createGoal
async createGoal(goalData, userId) { ... }

// ‚úÖ We have updateGoalProgress (updates current_value)
async updateGoalProgress(goalId, currentValue) { ... }

// ‚ùå We DON'T have updateGoalTarget
async updateGoalTarget(goalId, newTargetValue) { 
  // This function doesn't exist!
}
```

### **Should Users Be Able to Change Target?**

**Pros of Allowing Changes:**
- User realizes "10 matches is too easy" ‚Üí increase to 15
- User realizes "70% win rate is too hard" ‚Üí decrease to 60%
- Flexible goal adjustment based on performance

**Cons of Allowing Changes:**
- Feels like "cheating" or "moving the goalposts"
- Loses historical context of original goal
- Progress percentage changes suddenly

**Recommendation:**
- DON'T allow editing `target_value` after creation
- If user wants different target, create NEW goal
- Keep old goal as historical record

## üéØ Best Practices

### **1. Always Store target_value**
```typescript
// ‚úÖ GOOD
const newGoal = {
  category: 'Total Matches',
  target_value: 10,  // Store the target
  current_value: 0
};

// ‚ùå BAD
const newGoal = {
  category: 'Total Matches',
  // Missing target_value!
  current_value: 0
};
```

### **2. Validate target_value on Creation**
```typescript
async createGoal(goalData, userId) {
  // ‚úÖ Validate target is positive
  if (!goalData.target_value || goalData.target_value <= 0) {
    throw new Error('Target value must be positive');
  }
  
  // ‚úÖ Validate target is reasonable
  if (goalData.target_value > 10000) {
    throw new Error('Target value too large');
  }
  
  // Create goal
}
```

### **3. Use target_value in Progress Calculation**
```typescript
// ‚úÖ GOOD
const progress = (current / target) √ó 100;

// ‚ùå BAD - Hard-coded target
const progress = (current / 10) √ó 100;  // What if target is 20?
```

### **4. Check Completion Correctly**
```typescript
// ‚úÖ GOOD - Greater than or equal
const isComplete = current_value >= target_value;

// ‚ùå BAD - Only equal
const isComplete = current_value === target_value;  // What if current goes to 11?
```

## üîë Key Takeaways

1. **‚úÖ `target_value` is ESSENTIAL** - Can't function without it
2. **‚úÖ Stored in database** - Each goal has its own target
3. **‚úÖ Used for progress calculation** - `(current / target) √ó 100`
4. **‚úÖ Used for completion check** - `current >= target`
5. **‚úÖ User-specific** - Different users have different targets
6. **‚úÖ Goal-specific** - Same user can have different targets for different goals
7. **‚ùå Not editable** - Once set, target doesn't change (by design)
8. **‚úÖ Can be exceeded** - current_value can go above target_value
